# 🧪 ユーザビリティテストシナリオ

**狩猟免許試験 完全攻略アプリのユーザビリティテスト手順書**

作成日: 2026-02-06

---

## 📋 テストの目的

このドキュメントは、以下の目的でユーザビリティテストを実施するためのシナリオ集です：

- ユーザーが迷わずアプリを使えるか確認
- 典型的な学習フローがスムーズか検証
- 問題点・改善点を発見する
- 初心者でも一発合格を目指せるか確認

---

- 年齢: 28歳、会社員

- スマホ利用: 得意

- スマホ利用: 普通
- 学習時間: 週末に集中
- 目標: 弱点を克服して合格

### ペルソナC: シニア学習者
- 年齢: 65歳、退職者
- 狩猟免許試験: 初挑戦
- スマホ利用: 不慣れ
- 学習時間: 毎日1時間
- 目標: じっくり学習して確実に合格

## 📖 テストシナリオ

### シナリオ1: 初回アクセス（5分）

**目的**: アプリの第一印象、初期設定のスムーズさを確認

**手順**:
1. ブラウザでアプリのURLを開く
2. トップページを見る
   - **確認**: 何のアプリか一目で分かるか？
   - **確認**: 次に何をすべきか明確か？
3. 「アプリの使い方」を読む
   - **確認**: 理解できる内容か？
   - **確認**: 3分以内に読み終わるか？
4. PWAインストールを試す
   - **確認**: インストール方法が分かるか？
   - **確認**: ホーム画面に追加できるか？

**成功基準**:
- [ ] 5分以内に「次に何をすべきか」が理解できる
- [ ] PWAインストールに成功する
- [ ] 「学習を始めたい」と思える

---

### シナリオ2: 初めての問題演習（10分）

**目的**: 問題演習機能の使いやすさを確認

**手順**:
1. トップページから「問題演習」をクリック
2. カテゴリ「法令」を選択
3. 問題を5問解く
   - **確認**: 問題文が読みやすいか？
   - **確認**: 選択肢が押しやすいか？
   - **確認**: 正解・不正解が分かりやすいか？
4. 解説を読む
   - **確認**: 解説が理解しやすいか？
   - **確認**: 間違えた理由が分かるか？
5. 「次の問題」で進む
   - **確認**: スムーズに進めるか？

**成功基準**:
- [ ] 問題を解く操作が直感的
- [ ] 解説が分かりやすい
- [ ] 「もっと問題を解きたい」と思える

---

### シナリオ3: 鳥獣図鑑で識別学習（10分）

**目的**: 鳥獣図鑑の学習効果を確認

**手順**:
1. トップページから「鳥獣図鑑」をクリック
2. 「フラッシュカード」モードを試す
   - **確認**: 操作方法が分かるか？
   - **確認**: 5秒判別が面白いか？
3. 10種類の鳥獣を学習
   - **確認**: 特徴が覚えやすいか？
   - **確認**: 似た種との違いが分かるか？
4. 「狩猟鳥獣」と「非狩猟鳥獣」の違いを理解
   - **確認**: 区別が明確か？

**成功基準**:
- [ ] フラッシュカードが楽しく学べる
- [ ] 10種類のうち7種類以上を覚えられる
- [ ] 狩猟可否の判断ができるようになる

---

### シナリオ4: 模擬試験を受験（90分）

**目的**: 本番同様の試験体験、合格判定の精度を確認

**手順**:
1. トップページから「模擬試験」をクリック
2. 試験開始
   - **確認**: 制限時間が表示されるか？
   - **確認**: 残り時間が分かりやすいか？
3. 30問を解く
   - **確認**: 途中保存できるか？
   - **確認**: 見直しができるか？
4. 試験終了後、結果を確認
   - **確認**: 合格・不合格が明確か？
   - **確認**: 弱点カテゴリが分かるか？
5. 間違えた問題を復習
   - **確認**: 復習機能が使いやすいか？

**成功基準**:
- [ ] 本番試験と同じ緊張感を持てる
- [ ] 結果から次の学習方針が分かる
- [ ] 間違えた問題を効率的に復習できる

---

### シナリオ5: 学習ノートで復習（15分）

**目的**: 学習ノート機能の活用度を確認

**手順**:
1. トップページから「学習ノート」をクリック
2. 新しいノートを作成
   - **確認**: 入力が簡単か？
   - **確認**: カテゴリ選択が分かりやすいか？
3. 過去のノートを検索
   - **確認**: 検索機能が使いやすいか？
4. ノートをエクスポート
   - **確認**: エクスポート機能が分かるか？
   - **確認**: 他の端末で利用できるか？

**成功基準**:
- [ ] ノート作成が5分以内にできる
- [ ] 検索で必要なノートが見つかる
- [ ] エクスポート・インポートが成功する

---

### シナリオ6: 進捗ダッシュボードで現状把握（5分）

**目的**: 進捗管理機能の分かりやすさを確認

**手順**:
1. トップページから「進捗管理」をクリック
2. 総合正答率を確認
   - **確認**: 自分の実力が一目で分かるか？
3. カテゴリ別正答率を確認
   - **確認**: 弱点カテゴリが分かりやすいか？
4. 合格可能性予測を確認
   - **確認**: 信頼できる予測か？
5. 学習時間の記録を確認
   - **確認**: モチベーション向上につながるか？

**成功基準**:
- [ ] 現状の実力が正確に把握できる
- [ ] 次に何を学習すべきか明確になる
- [ ] 学習のモチベーションが上がる

---

### シナリオ7: ゲームモードで楽しく学習（10分）

**目的**: ゲーミフィケーション要素の効果を確認

**手順**:
1. トップページから「ゲームモード」をクリック
2. クイックマッチ（5問）を開始
   - **確認**: ゲーム感覚で楽しめるか？
3. 連続正解でコンボを狙う
   - **確認**: コンボシステムが分かりやすいか？
   - **確認**: 達成感があるか？
4. バッジを獲得
   - **確認**: バッジが嬉しいか？
5. ランキングを確認
   - **確認**: 他のユーザーと競える楽しさがあるか？

**成功基準**:
- [ ] ゲーム感覚で楽しく学習できる
- [ ] バッジやランキングがモチベーションになる
- [ ] 「もう1回やりたい」と思える

---

### シナリオ8: 3週間プランで計画的学習（3週間）

**目的**: 3週間学習プランの実用性を確認

**手順**:
1. トップページから「学習ガイド」をクリック
2. 3週間プランを確認
   - **確認**: 計画が具体的か？
   - **確認**: 実行可能なスケジュールか？
3. 1週目のタスクを実行
   - Day 1: 基礎知識の習得
   - Day 2: 法令の学習
   - ...
   - **確認**: 毎日のタスクが明確か？
4. 2週目、3週目も継続
   - **確認**: 無理なく続けられるか？
5. 模擬試験で最終確認
   - **確認**: 合格レベルに到達したか？

**成功基準**:
- [ ] 3週間継続できる
- [ ] 計画通りに学習が進む
- [ ] 最終的に合格レベルに到達する

## 🔍 アクセシビリティテスト

### キーボード操作テスト（10分）

**手順**:
1. マウスを使わずにTabキーで全ての機能を操作
   - **確認**: 全ての機能に到達できるか？
   - **確認**: フォーカスが分かりやすいか？
2. Enterキーで決定、Escキーでキャンセル
   - **確認**: 直感的に操作できるか？

**成功基準**:
- [ ] キーボードだけで全機能が使える
- [ ] フォーカスインジケーターが明確

### スクリーンリーダーテスト（15分）

**手順**:
1. VoiceOver（iOS/Mac）またはTalkBack（Android）を起動
2. 全ページを音声読み上げで確認
   - **確認**: 内容が理解できるか？
   - **確認**: ARIA属性が適切か？

**成功基準**:
- [ ] スクリーンリーダーで全ページが理解できる
- [ ] 操作が分かりやすい

## 🎯 合格基準

### 総合評価基準

| 項目 | 基準 |
|------|------|
| **使いやすさ** | 初心者が30分以内に操作方法を理解できる |
| **学習効果** | 3週間プランで合格レベルに到達できる |
| **アクセシビリティ** | WCAG 2.1 AA準拠、スクリーンリーダー対応 |
| **パフォーマンス** | ページ読み込み3秒以内 |
| **安定性** | エラーが発生しても適切に復旧できる |

### 総合判定

- **合格**: 全シナリオで成功基準を満たす
- **条件付き合格**: 軽微な問題はあるが、学習に支障なし
- **不合格**: 重大な問題があり、改善が必要
